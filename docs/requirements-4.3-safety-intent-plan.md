# 4.3 安全意图校验器开发与验收方案

本文对应 `docs/requirements.tex` 第 4.3 节 FR-3.1 至 FR-3.17。安全意图校验器位于
LLM 工具计划与实际工具执行之间，必须在 `PermissionEnforcer` 之前给出独立风险结论；权限足够
不等于操作安全，L4 操作即使处于最高权限模式也必须拒绝。

## 1. 执行链与复用边界

```text
LLM tool calls
    |
    v
StructuredIntentParser          FR-3.1..3.3
    |
    v
RiskEngine + HotRuleStore       FR-3.4..3.10
    |
    +-- L1: allow
    +-- L2: allow + audit
    +-- L3: confirmation gate   FR-3.11..3.14
    +-- L4: deny
    |
    v
SessionRiskTracker              FR-3.15..3.17
    |
    v
PermissionEnforcer -> tool/plugin/MCP execution
```

优先复用以下 Claw Code 组件：

- `runtime::bash_validation`：命令语义、只读、破坏性和路径检查的已有分类依据。
- `runtime::PermissionEnforcer`：风险校验通过后的最小权限门禁，不承担 L4 豁免。
- `runtime::ApprovalTokenLedger`：将 L3 确认绑定到操作摘要、操作者、执行者和过期时间。
- `runtime::LeastPrivilegeSession`：会话暂停时冻结权限，恢复后仍按租约和作用域执行。
- `runtime::ConversationRuntime`：在 pre-tool hook 后、实际 ToolExecutor 前接入风险校验。
- `plugins` 的动作、回滚和不可逆元数据：生成影响范围与回滚建议。

## 2. 需求逐项方案

| 需求 | 开发方案 | 验收重点 |
| --- | --- | --- |
| FR-3.1 | 将每个结构化 tool call 转换为 `StructuredIntent`，包含动作、目标、参数摘要、影响范围、隐含操作及来源。 | 不依赖自然语言猜测；未知字段和超限输入 fail closed。 |
| FR-3.2 | 多 tool plan 按原顺序拆分；shell 类工具保守解析明确的复合分隔符，引号内分隔符不得误拆。 | `&&`/`;`/换行后的危险尾部不能遗漏；不完整引号直接进入高不确定性。 |
| FR-3.3 | 由 tool/action 映射隐含能力，例如包安装同时需要网络、文件写入和进程执行。 | 隐含操作进入评分证据，不只用于展示。 |
| FR-3.4 | 规则引擎对每条 intent 输出 L1/L2/L3/L4 与 allow/audit/confirm/deny。 | 相同输入和规则版本产生确定结果；未知意图不默认放行。 |
| FR-3.5 | 分别计算命令类型、目标路径、参数危险度和影响范围四因子，再按权重汇总。 | 输出分项分数、命中规则和总分，便于审计解释。 |
| FR-3.6 | 严格反序列化管理员配置，支持权重和阈值调整；非法配置不替换当前健康版本。 | 权重有限且非负、总和有效、阈值单调；硬性 L4 不可降级。 |
| FR-3.7 | 维护带稳定 ID、严重级别、模式和版本的危险命令规则库。 | 覆盖格式化、块设备覆写、fork bomb、权限越界和凭据读取。 |
| FR-3.8 | 维护敏感路径保护集合，写入 `/boot`、`/etc/fstab`、账户/提权文件、系统二进制和内核接口默认拒绝。 | 规范化路径后匹配，防止前缀混淆、符号链接和大小写绕过。 |
| FR-3.9 | 检测递归/通配符删除、路径遍历、选项注入、shell 展开、重定向和编码绕过。 | 不以简单 substring 代替参数边界；疑似绕过至少升级为 L3。 |
| FR-3.10 | 使用不可变规则快照和单调 generation 热更新；完整校验成功后原子替换。 | 并发调用只看到旧或新完整快照；失败更新保留旧版本。 |
| FR-3.11 | L3 返回结构化确认请求，只有明确的 allow-once/deny 决策才能继续。 | 普通“继续”“好的”等非结构化文本不能被当作确认。 |
| FR-3.12 | 确认内容包含操作、目标与影响范围、风险分项、命中规则、回滚建议和不可逆说明。 | 对敏感参数做摘要和脱敏，不回显密钥。 |
| FR-3.13 | 默认 60 秒确认租约，使用单调时间或显式时钟；过期、重放和 scope 不匹配均拒绝。 | 超时后不能靠延迟响应恢复旧授权。 |
| FR-3.14 | 批量模式支持逐条确认；批量确认令牌绑定完整 plan 哈希和每项风险，计划变化后失效。 | 批量确认不得扩展到新操作；包含 L4 时整体仍拒绝。 |
| FR-3.15 | 会话记录时间窗内的 intent、目标、风险分和结果，计算累积风险。 | 重复删除、批量账户/服务变更等会提升分数。 |
| FR-3.16 | 用状态化序列规则检测移动后删除、复制敏感文件后外传、分步降权/提权等模式。 | 规则只基于已规范化意图和会话事件，给出关联证据。 |
| FR-3.17 | 达到暂停阈值后冻结会话执行和权限升级，生成审核快照；只有新的审核令牌可恢复。 | 暂停后 read/write/plugin/MCP 均不能绕过；恢复动作有审计记录。 |

## 3. 风险模型

四个因子统一输出 0 至 100：

- `command_type`：只读查询最低，状态变更居中，破坏性/系统管理最高。
- `target_path`：工作区普通文件较低，系统配置、凭据、引导、设备和内核接口最高。
- `parameter_danger`：递归、force、通配符、权限开放、shell 展开和路径遍历逐级加分。
- `impact_scope`：单文件/单服务、目录/一组服务、全局/递归/批量依次升高。

总分使用管理员权重归一化计算，并按单调阈值映射等级。以下规则不经过加权降级：

- 格式化块设备、覆盖磁盘或引导区。
- 删除根目录、系统目录或无边界递归删除。
- fork bomb、内核/引导破坏及明确的凭据批量导出。
- 已命中的强制拒绝规则，或解析器确认无法完整覆盖的高危复合命令。

风险结果必须包含 `ruleGeneration`、输入摘要哈希、四因子证据、最终策略和是否由硬规则覆盖，
以便审计时重放同一规则版本。

## 4. 规则热更新

规则文件使用版本化 JSON，限制文件大小、规则数量、正则长度和编译时间。加载流程为：

1. 从受信配置目录读取普通文件，拒绝符号链接/reparse point。
2. 严格反序列化并检查 schemaVersion、generation、稳定规则 ID 和重复项。
3. 编译模式并运行内置安全自检样例，确保硬性 L4 仍存在。
4. 使用内容哈希和管理员签名/受信文件权限确认来源。
5. 原子替换 `Arc` 快照；记录成功或失败审计事件。

规则更新只允许增加或调整可配置评分规则，不能删除编译进程序的硬性拒绝底线。

## 5. 确认与批量执行

L3 确认请求生成一次性 scope：`sessionId + planHash + intentHash + ruleGeneration + expiresAt`。
单项模式逐个消费令牌；批量模式对有序 intent 哈希列表签发一个令牌。任何参数、顺序、规则代次或
操作者变化都会导致 scope 不匹配。

确认展示数据来自结构化风险结果而非重新拼接原始命令。回滚建议优先读取插件显式 rollback 元数据；
没有确定性回滚时必须标记不可逆，不能生成虚假的恢复承诺。

## 6. 会话关联分析

`SessionRiskTracker` 只保留有界事件窗口和摘要哈希，避免把敏感参数长期写入会话。累积模型包括：

- 时间衰减后的风险分总和。
- 同一动作/目标族的频率与批量规模。
- 跨步骤攻击序列状态，例如 move -> delete、read-secret -> network-send。
- 已拒绝操作和随后轻微变形重试的惩罚分。

达到阈值时状态转为 `PausedForReview`，后续风险校验先返回暂停错误，再进入权限/工具层。恢复令牌
必须绑定审核快照哈希，并清楚记录 reviewer、理由和新的风险预算。

## 7. 验收计划

通用测试至少覆盖：

- 结构化单操作、多操作和 shell 复合操作拆分。
- quoted delimiter、转义、畸形引号和危险尾部。
- 四因子评分、隐含操作证据、配置边界和强制 L4。
- 危险命令、敏感路径、遍历、通配符、选项注入及热更新并发。
- L3 允许、拒绝、超时、重放、scope 变化和批量计划变化。
- 会话累积分、攻击序列、暂停、跨工具绕过和审核恢复。
- conversation -> safety intent -> permission -> tool 的端到端拒绝/确认/执行链。

麒麟目标机补充真实 `bash`、systemd、dnf、firewalld 和插件调用测试；Windows 只能证明平台无关
解析、评分和 fail-closed 行为，不能替代麒麟命令及权限集成验收。
